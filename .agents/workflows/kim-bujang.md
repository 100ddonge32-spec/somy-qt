# 🐑 소미(Somy) 프로젝트 - 김부장 가이드라인

이 워크플로우는 '김부장'의 개발 철학과 [소미 QT] 프로젝트의 핵심 운영 방식을 정의합니다.

### 1. 📋 김부장의 개발 철학
- **깔끔함(Clarity)**: 사용자 화면에는 꼭 필요한 정보만 노출한다. 불필요한 레이아웃은 과감히 생략한다.
- **성도 중심(User-Centric)**: 성경 문체는 '따뜻한 목회자'의 톤앤매너를 유지한다.
- **풍성한 묵상**: 성경 본문은 너무 짧지도(1절), 너무 길지도(전체 장) 않은 **약 15~20절** 분량이 가장 적절하다.

### 2. 📖 본문 관리 및 생성 (Core Logic)
- **자동 생성**: `src/app/api/qt/route.ts`에서 AI가 성경 읽기표에 따라 본문을 추출할 때, 20절 제한 로직이 잘 작동하는지 항상 확인한다.
- **수동 관리**: 관리자 모드(⚙️)에서 '오늘 본문 불러오기'를 실행하여 내용을 검토한 뒤 저장한다.

### 3. 🚀 배포 워크플로우
변경사항이 발생하면 반드시 아래 순서로 배포를 진행한다:
1. `cd "/Users/macbook/어플 개발 테스트/somy-qt"`
2. `git add .`
3. `git commit -m "작업 내용 요약"`
4. `git push` (Vercel 자동 배포 확인)

### 4. 🛠️ 문제 해결 가이드
- **본문이 안 바뀔 때**: DB의 `plan` 컬럼이 `premium`인지 확인하고, 관리자 모드에서 '불러오기'를 다시 실행한다.
- **로그인 오류**: 카카오/구글 API 키와 Supabase 세션 상태를 체크한다.
